Fail Soft

A network security strategy that allows a system to continue functioning in a limited capacity after a failure or breach.

Description

Fail soft is a design philosophy in network security that prioritizes system availability and usability even in the event of a partial failure or security breach. Instead of failing completely and shutting down operations, a fail soft system will attempt to limit the impact of the failure while still allowing certain functionalities to remain available. This approach is particularly valuable in environments where uptime is critical, such as in financial services or healthcare. For instance, if a network intrusion detection system (IDS) identifies a potential threat, instead of shutting down the entire network, it may isolate the affected components or restrict access to sensitive data while keeping other services operational. This way, organizations can mitigate risks without completely disrupting their services, allowing them to maintain some level of operation and continue to serve users, albeit in a restricted manner. Fail soft strategies often involve implementing redundancy, graceful degradation, and robust incident response procedures to ensure that critical functions can persist even under duress.

Examples

  • A web application that limits user access instead of going offline entirely during a security incident.
  • A database that continues to serve read requests while rejecting write requests when it detects a potential corruption.

Additional Information

  • Fail soft can help in maintaining user trust during incidents.
  • It is often contrasted with 'fail secure' systems that prioritize security over availability.
