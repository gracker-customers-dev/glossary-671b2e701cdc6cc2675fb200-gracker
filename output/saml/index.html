SAML

Security Assertion Markup Language, an open standard for exchanging authentication and authorization data.

Description

SAML (Security Assertion Markup Language) is an open standard for exchanging authentication and authorization data between parties, specifically between an identity provider and a service provider. It enables Single Sign-On (SSO) by allowing users to authenticate once and gain access to multiple applications without needing to log in again. SAML uses XML-based assertions to transfer user identity data and attributes from the identity provider to the service provider. This process involves several key components: the user (or principal), the identity provider (IdP), and the service provider (SP). When a user attempts to access a service provider, they are redirected to the IdP for authentication. Upon successful authentication, the IdP sends a SAML assertion back to the SP, which grants access based on the information contained in the assertion. SAML is widely used in enterprise environments for federated identity management, where organizations need to manage user identities across various systems securely.

Examples

  • Using SAML for logging into cloud applications like Salesforce or Google Workspace.
  • Integrating SAML-based SSO in corporate environments for internal applications.

Additional Information

  • SAML is maintained by OASIS (Organization for the Advancement of Structured Information Standards).
  • It supports multiple authentication methods, including username/password, MFA, and more.
