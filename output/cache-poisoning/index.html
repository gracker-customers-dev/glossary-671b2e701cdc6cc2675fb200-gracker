Cache Poisoning

A network attack method that involves injecting false data into a cache to manipulate the responses received by clients.

Description

Cache poisoning is a type of attack that targets the caching mechanisms used in various network protocols, notably DNS (Domain Name System) and web caching systems. In a typical cache poisoning attack, the attacker sends forged responses to a cache server, causing it to store incorrect data. This poisoned data can then be served to legitimate users, redirecting them to fraudulent websites, intercepting sensitive information, or spreading malware. The attack exploits the trust that systems place in cached data, undermining the integrity of the information stored for quick access. For example, if a DNS server is poisoned, users attempting to reach a legitimate site may be redirected to a malicious counterpart. Cache poisoning can also occur in web proxies, where a cached page may serve outdated or harmful content to users. This threat emphasizes the importance of validating and authenticating data before it is cached, alongside implementing security measures to mitigate such vulnerabilities.

Examples

  • DNS Cache Poisoning: An attacker sends a forged DNS response to a resolver, redirecting users from a legitimate website to a malicious one.
  • Web Cache Poisoning: An attacker manipulates cached web content, causing users to receive altered or harmful web pages.

Additional Information

  • Cache poisoning can lead to phishing attacks, data theft, and the spread of malware.
  • Mitigation strategies include using DNSSEC for DNS queries and implementing strict validation on cached data.
