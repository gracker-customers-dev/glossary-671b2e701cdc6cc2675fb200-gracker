Port Address Translation (PAT)

PAT is a technique used in network security that allows multiple devices on a local network to be mapped to a single public IP address but with a different port number for each session.

Description

Port Address Translation (PAT), often referred to as NAT overload, is a form of Network Address Translation (NAT) that enables multiple devices on a private network to access external networks using a single public IP address. When a device on the internal network initiates a connection to the internet, PAT modifies the source IP address of the outgoing packets to the public IP address of the router, and it assigns a unique port number to each session. This way, the router can keep track of which internal device initiated the connection by mapping the original private IP address and port number to the public IP address and the unique port number. When the response packets return from the external network, PAT uses the unique port numbers to route the packets back to the correct internal device. This technique not only conserves the number of public IP addresses needed but also provides a degree of security by hiding the internal IP addresses from external networks.

Examples

  • A home router that allows multiple devices (like phones and laptops) to access the internet using one public IP address.
  • An organization using PAT to enable employees' computers to connect to the internet while maintaining a single public IP for external services.

Additional Information

  • PAT is commonly used in conjunction with firewalls to enhance network security.
  • While PAT helps with IP address conservation, it can complicate certain applications like VoIP, which may require direct IP addressing.
