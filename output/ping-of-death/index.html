Ping of Death

A type of network attack that involves sending malicious ICMP packets to crash or disrupt a target system.

Description

The Ping of Death is a well-known network attack that exploits the Internet Control Message Protocol (ICMP). It involves sending oversized ping packets—specifically, ICMP Echo Request packets—that exceed the maximum allowable size for a single packet, which is 65,535 bytes. When the target system receives these malformed packets, it may not handle them properly, potentially causing it to crash, freeze, or reboot. This vulnerability was particularly prevalent in older operating systems that did not have adequate protections against such oversized packets. The Ping of Death attack highlights the importance of robust network security measures and the need for proper packet filtering and size validation on network devices. Although modern operating systems have largely mitigated this vulnerability, the concept remains an important lesson in network security, illustrating how the misuse of legitimate network protocols can lead to denial-of-service conditions.

Examples

  • An attacker sends a malformed ICMP packet that exceeds the size limit to a vulnerable Windows 95 machine, causing it to crash.
  • A network administrator configures a firewall to block oversized ICMP packets to prevent potential Ping of Death attacks.

Additional Information

  • The Ping of Death exploit was most effective against systems with outdated or unpatched software.
  • Mitigation strategies include updating systems, configuring firewalls to filter out oversized packets, and implementing intrusion detection systems.
