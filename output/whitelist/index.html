Whitelist

A list of entities that are granted access or permission within a network or system.

Description

In network security, a whitelist is a predefined list of entities such as IP addresses, email addresses, domain names, or applications that are considered safe and are allowed to access a network or perform certain actions. Unlike blacklists, which block known threats, whitelists operate on a trust-based model where only the approved entities are permitted while all others are denied access by default. This approach enhances security by significantly reducing the attack surface, as it limits the potential entry points for malicious activities. Whitelisting can be applied to various contexts, including application whitelisting, where only approved applications can run on a system, and network whitelisting, where only whitelisted IP addresses can connect to the network. The implementation of whitelists requires careful management, as legitimate entities must be continuously evaluated to ensure they remain trusted, and adjustments must be made to the list as necessary to accommodate new or changing requirements.

Examples

  • An organization uses a whitelist of IP addresses to restrict access to its internal network.
  • A company implements application whitelisting to ensure that only authorized software can execute on employee computers.

Additional Information

  • Whitelisting can improve system performance by reducing the number of security checks performed on approved entities.
  • Maintaining an up-to-date whitelist is crucial, as outdated entries can lead to operational disruptions.
