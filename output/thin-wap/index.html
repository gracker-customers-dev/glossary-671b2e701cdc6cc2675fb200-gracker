Thin WAP

A Thin Wireless Access Point (WAP) is a network device that connects wireless clients to a wired network but relies on a central controller for management and configuration.

Description

Thin WAPs are designed to simplify network management by offloading most of the processing and control functions to a centralized controller, which can be located on-premises or in the cloud. This architecture allows for easier scalability, as adding new access points can often be done with minimal configuration. Thin WAPs typically do not handle tasks such as authentication and security at the device level; instead, these functions are managed centrally by the controller. This setup is particularly beneficial in large environments, such as corporate offices, schools, and other institutions, where multiple access points are needed to provide adequate coverage and capacity. The central controller can enforce security policies, manage firmware updates, and monitor network performance, ensuring that the entire wireless network operates efficiently and securely. This model contrasts with fat access points, which manage their own configurations and processes independently.

Examples

  • Cisco Lightweight Access Points managed by Cisco Wireless Controllers.
  • Aruba Instant Access Points that can be centrally managed by Aruba Central.

Additional Information

  • Thin WAPs typically require a robust network infrastructure to support the centralized controller.
  • They are often used in environments where consistent security policies and configurations are critical.
