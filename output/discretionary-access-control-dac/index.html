Discretionary Access Control (DAC)

A type of access control where the owner of a resource decides who can access it.

Description

Discretionary Access Control (DAC) is a security model that allows resource owners to control access to their resources based on their discretion. In a DAC system, users have the ability to grant or restrict access rights to other users, which can lead to a flexible but potentially less secure environment. This model contrasts with Mandatory Access Control (MAC), where access rights are assigned by a central authority and cannot be modified by the owners. DAC is often implemented in operating systems and database management systems, allowing users to set permissions for files, folders, and other resources. For example, a user may choose to share a document with specific colleagues while restricting access to others. While DAC provides ease of use and flexibility, it may also lead to security vulnerabilities if users inadvertently grant access to unauthorized individuals. Proper training and awareness are essential to mitigate these risks, ensuring that users understand the implications of the permissions they assign.

Examples

  • A user sharing a file on a cloud storage platform with selected individuals.
  • A database administrator granting access privileges to specific users for certain tables.

Additional Information

  • DAC can lead to accidental data exposure if users are not careful with permissions.
  • It is commonly used in systems like Windows and Unix/Linux, where file permissions are set by the file owner.
