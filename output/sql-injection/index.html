SQL Injection

A code injection technique that exploits vulnerabilities in an application’s software by manipulating SQL queries.

Description

SQL Injection (SQLi) is a type of attack that allows an attacker to interfere with the queries that an application makes to its database. This generally occurs when an application does not properly sanitize user input, allowing an attacker to inject arbitrary SQL code. By exploiting this vulnerability, attackers can gain unauthorized access to the database, retrieve sensitive data, modify or delete records, and in some cases, execute administrative operations on the database. SQL Injection can range from simple attacks, such as retrieving user credentials, to complex attacks that escalate privileges or compromise the entire system. It is one of the most common web application vulnerabilities and can have severe consequences, including data breaches, loss of data integrity, and damage to the application's reputation. To mitigate SQL Injection risks, developers should use prepared statements, parameterized queries, and ORM frameworks, along with robust input validation and sanitization techniques.

Examples

  • A user input field that allows entering a username and password could be exploited with an input like ' OR '1'='1' -- to bypass authentication.
  • An attacker could manipulate a search query to execute a command such as 'SELECT * FROM users WHERE username = '' UNION SELECT credit_card_number, expiration_date FROM credit_cards --' to extract sensitive information.

Additional Information

  • SQL Injection is listed as one of the OWASP Top Ten web application vulnerabilities.
  • Regular security audits and code reviews can help identify and fix SQL Injection vulnerabilities.
