Message Authentication Code (MAC)

A MAC is a short piece of information used to authenticate a message and confirm its integrity.

Description

A Message Authentication Code (MAC) is a cryptographic checksum on data that provides both integrity and authenticity. It is generated by applying a secret key to the data using a cryptographic hash function or block cipher. The resulting MAC is sent along with the message to the recipient, who can then compute the MAC on the received message using the same secret key. If the computed MAC matches the MAC included with the message, the recipient can be confident that the message has not been altered in transit and that it comes from a legitimate sender. MACs are used in various protocols and applications, such as TLS (Transport Layer Security), wireless communication standards, and secure messaging systems. They help protect against forgery and tampering, ensuring that the data remains intact and authenticated during transmission. Unlike digital signatures, MACs require both the sender and receiver to share a secret key, making them suitable for scenarios where both parties are trusted to some extent.

Examples

  • HMAC (Hash-based Message Authentication Code) - a widely used MAC that employs a hash function and a secret key.
  • CMAC (Cipher-based Message Authentication Code) - a MAC that uses a block cipher for generating the authentication code.

Additional Information

  • MACs are used in various network protocols including IPsec and SSH.
  • The security of a MAC relies on the strength of the underlying cryptographic algorithm and the secrecy of the key.
