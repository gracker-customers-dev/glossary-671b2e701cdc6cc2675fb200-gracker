Server-Side Request Forgery (SSRF)

A type of security vulnerability that allows an attacker to induce a server to make unintended requests to other internal or external resources.

Description

Server-Side Request Forgery (SSRF) is a security vulnerability that occurs when a web application or server is tricked into making HTTP requests on behalf of an attacker. Typically, SSRF exploits arise when a server accepts user input to fetch data from a given URL without proper validation and sanitization. This allows attackers to manipulate the input to make the server request resources that it should not access, such as internal APIs, databases, or other services. For example, an attacker could exploit an SSRF vulnerability to access sensitive metadata in cloud environments or to enumerate internal network services that are not exposed to the public internet. As a result, SSRF can lead to unauthorized information disclosure, denial of service, or even remote code execution if the server is misconfigured. Mitigating SSRF vulnerabilities involves validating and sanitizing user input, implementing strict access controls, and using whitelisting techniques to limit the URLs the server can request.

Examples

  • An attacker exploits SSRF to access internal AWS metadata endpoints to retrieve sensitive information such as IAM credentials.
  • A web application that fetches thumbnails from user-provided URLs is exploited by an attacker to access internal services that should not be publicly accessible.

Additional Information

  • SSRF vulnerabilities are often found in applications that process user-supplied URLs, such as image upload services, URL shorteners, or webhooks.
  • Preventing SSRF requires careful design, including implementing network segmentation and limiting outbound connections from servers.
