Deny by Default

A security principle that assumes all traffic is untrusted unless explicitly allowed.

Description

Deny by Default is a fundamental security principle in network security that dictates that, by default, all incoming and outgoing network traffic should be denied unless there are specific rules or policies that explicitly allow that traffic. This approach helps to minimize the attack surface by ensuring that only necessary services are exposed and that unauthorized access is prevented. In practice, this means that firewall rules and access control lists (ACLs) should be configured to deny all traffic initially, and then specific exceptions should be made for trusted sources or necessary services. This principle is often contrasted with 'allow by default', where all traffic is permitted unless explicitly blocked. By adopting a Deny by Default stance, organizations can enhance their security posture by preventing potential threats and vulnerabilities from being exploited. It requires careful planning and management of network policies to ensure that legitimate traffic is not inadvertently blocked, while still maintaining a strong security framework.

Examples

  • A firewall is configured to block all incoming traffic unless specific ports are opened for services like HTTP or HTTPS.
  • An organization's server only allows connections from a predefined list of IP addresses, denying all other traffic by default.

Additional Information

  • Deny by Default is often implemented in conjunction with the principle of least privilege.
  • Regular audits and reviews of access controls are necessary to ensure compliance with Deny by Default policies.
