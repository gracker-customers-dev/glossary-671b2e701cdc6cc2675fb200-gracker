DNS based Phishing

A type of phishing attack that exploits vulnerabilities in the Domain Name System (DNS) to redirect users to malicious websites.

Description

DNS based phishing is a malicious technique where attackers manipulate the Domain Name System to redirect users attempting to access legitimate websites to fake ones that resemble the original sites. This is typically done by hijacking DNS records or using techniques like DNS spoofing to alter the resolution of domain names. When a user types in a web address, the DNS query is sent out to resolve the domain to an IP address. An attacker can intercept this query and respond with a false IP address, leading the user to a fraudulent website designed to steal sensitive information, such as usernames, passwords, or credit card details. This method is particularly effective because many users trust the URL they see in their browser and may not notice subtle differences in the domain name. To protect against DNS based phishing, users and organizations should implement security measures such as DNSSEC (Domain Name System Security Extensions) and regularly monitor for suspicious domain registrations and DNS changes.

Examples

  • A user tries to access their bank's website but is redirected to a fake site that looks identical to the original.
  • An employee receives an email with a link that appears to be from a trusted source but redirects them to a malicious domain due to DNS manipulation.

Additional Information

  • DNSSEC helps to secure DNS by allowing DNS responses to be verified for authenticity.
  • Phishing attacks can be mitigated through user education and awareness training to recognize fraudulent websites.
