Credential stuffing

A type of cyber attack where attackers use stolen username and password pairs to gain unauthorized access to user accounts.

Description

Credential stuffing is a specific form of cyber attack that exploits the tendency of users to reuse passwords across multiple sites. In this attack, cybercriminals obtain a list of compromised credentials, often from data breaches, and then use automated tools to systematically attempt to log into various websites and services using these credentials. Because many users recycle their passwords, successful logins can occur even when the credentials are not intended for the target site. This practice poses significant risks to both users and organizations, as it can lead to account takeovers, data breaches, and unauthorized transactions. Organizations can mitigate this risk by implementing multi-factor authentication, monitoring for unusual login patterns, and educating users about the importance of unique, complex passwords for different accounts. The prevalence of credential stuffing has been increasing as more and more user credentials become available on the dark web, making it a critical concern for cybersecurity professionals.

Examples

  • An attacker uses a list of stolen credentials from a major data breach to access accounts on a popular e-commerce site.
  • Cybercriminals employ bots to test thousands of username/password combinations on a social media platform to gain unauthorized access.

Additional Information

  • Credential stuffing is often facilitated by the use of automated tools and bots that can quickly test many combinations.
  • Organizations can defend against credential stuffing by implementing rate limiting, CAPTCHA, and anomaly detection on login attempts.
