SYN Flood

A type of Denial of Service (DoS) attack that exploits the TCP handshake process.

Description

A SYN flood attack is a form of Denial of Service (DoS) attack that targets the Transmission Control Protocol (TCP) handshake. In a typical TCP connection establishment, a client initiates a connection to a server by sending a SYN (synchronize) packet. The server responds with a SYN-ACK (synchronize-acknowledge), and the client then replies with an ACK (acknowledge) packet to complete the handshake. In a SYN flood attack, the attacker sends a large number of SYN packets to the server, often using spoofed IP addresses. The server allocates resources and waits for the corresponding ACK packets that never arrive, leading to an overload of half-open connections. As the server exhausts its resources, legitimate users are unable to establish connections, causing a Denial of Service. This type of attack can severely impact the availability of services and is often used in conjunction with other attacks to disrupt network operations.

Examples

  • An attacker sends thousands of SYN packets to a web server, overwhelming its ability to respond to legitimate requests.
  • A botnet is used to perform a coordinated SYN flood on multiple servers, amplifying the impact of the attack.

Additional Information

  • SYN flood attacks can be mitigated using SYN cookies, which help to manage resource allocation without maintaining state for each connection.
  • Firewall and intrusion detection systems can be configured to detect and block SYN flood patterns.
