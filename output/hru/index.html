HRU

HRU stands for 'Harrison-Ruzzo-Ullman' model, which is a formal model for access control in computer security.

Description

The HRU model, developed by Michael Harrison, Walter Ruzzo, and Jeffrey Ullman in the 1970s, is a foundational framework for understanding access control in computer systems. It provides a formal way to analyze and reason about the security properties of systems with respect to access rights and permissions. The model includes concepts such as subjects (users or processes) and objects (files or resources) and defines access rights that subjects can have over objects. It also introduces the concept of a 'state' of the system, which includes the current access rights and the state of the subjects and objects. One of the key contributions of the HRU model is its focus on ensuring that the system maintains security properties, such as the absence of unauthorized access, upon the execution of certain operations (like granting or revoking access rights). This model serves as a basis for various access control mechanisms and policies used in modern operating systems and security protocols.

Examples

  • Implementation of access control lists (ACLs) in operating systems.
  • Formal verification of security properties in distributed systems using the HRU model.

Additional Information

  • The HRU model is an important part of the study of formal methods in computer security.
  • It has influenced many contemporary access control models, such as Role-Based Access Control (RBAC).
