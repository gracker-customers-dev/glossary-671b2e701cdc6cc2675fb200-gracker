Address Resolution Protocol (ARP)

A protocol used to map IP addresses to MAC addresses in a local area network.

Description

Address Resolution Protocol (ARP) is a network protocol used to discover the link layer address, such as a Media Access Control (MAC) address, associated with a given Internet Protocol (IP) address. ARP operates at the network layer of the Internet Protocol Suite and is crucial for enabling communication within a local area network (LAN). When a device wants to communicate with another device on the same network but only knows its IP address, it broadcasts an ARP request to all devices on the network. The device with the matching IP address responds with its MAC address, allowing the original device to create a link-layer frame to initiate communication. While ARP is essential for network operations, it is also susceptible to various security threats, such as ARP spoofing or ARP poisoning, where an attacker sends false ARP messages to associate their MAC address with the IP address of another device, potentially leading to data interception or denial of service attacks. Therefore, understanding ARP is crucial in the context of network security.

Examples

  • A computer on a network sends an ARP request to find the MAC address of a printer using its IP address.
  • A router uses ARP to resolve the MAC address of a device when forwarding packets within a LAN.

Additional Information

  • ARP operates on the OSI model's link layer and is defined in RFC 826.
  • Security measures such as Dynamic ARP Inspection (DAI) can help mitigate ARP spoofing attacks.
