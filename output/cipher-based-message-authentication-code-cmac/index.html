Cipher-based Message Authentication Code (CMAC)

CMAC is a message authentication code based on a symmetric key block cipher.

Description

Cipher-based Message Authentication Code (CMAC) is a cryptographic algorithm used to ensure the integrity and authenticity of a message. It operates using a symmetric key block cipher, such as AES or DES, to generate a fixed-size output known as a MAC (Message Authentication Code). CMAC takes an input message and a secret key, processes them through the cipher algorithm, and produces a unique MAC that can be used to verify that the message has not been altered and that it comes from a legitimate source. The primary advantage of CMAC over other MAC algorithms is its ability to provide security against certain types of attacks, including those targeting the integrity of the message. CMAC is widely used in various security protocols and applications, such as in the IEEE 802.11 wireless networking standard and in secure communications protocols like TLS. Its strength relies on the underlying block cipher used, making it crucial to select a robust cipher to ensure the security of the MAC generated.

Examples

  • CMAC is used in the IEEE 802.11i standard for wireless security.
  • CMAC is employed in the TLS protocol for securing web communications.

Additional Information

  • CMAC can be used with any block cipher, but AES is the most common choice.
  • CMAC is designed to work efficiently with variable-length messages.
