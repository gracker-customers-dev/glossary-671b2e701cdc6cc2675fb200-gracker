Stream Cipher

A method of encrypting data where each bit of plaintext is encrypted one at a time with a corresponding bit from a pseudorandom key stream.

Description

A stream cipher is a type of symmetric key cipher that encrypts data one bit or byte at a time. Unlike block ciphers, which encrypt fixed-size blocks of data, stream ciphers work on individual bits of data, making them suitable for scenarios where data arrives in a continuous stream, such as in telecommunications. The encryption process involves combining the plaintext data with a key stream generated from a secret key using a pseudorandom number generator. This combination is typically done using the XOR operation, where each bit of plaintext is XORed with a bit from the key stream to produce the ciphertext. The key stream must be as long as the plaintext and ideally should be unpredictable to ensure security. Stream ciphers are often faster and require less memory than block ciphers, making them ideal for resource-constrained environments. However, if the same key stream is reused, it can lead to vulnerabilities, so proper key management is essential.

Examples

  • RC4
  • Salsa20

Additional Information

  • Stream ciphers are generally faster than block ciphers for encrypting data streams.
  • They are often used in applications like SSL/TLS for secure internet communications.
