Clickjacking

A web-based attack where a user is tricked into clicking on something different from what the user perceives, potentially compromising their security.

Description

Clickjacking is a type of malicious technique used by attackers to deceive users into clicking on an element that is hidden or disguised on a webpage. This is often achieved by overlaying an iframe on a legitimate webpage, making it difficult for the user to realize they are interacting with the hidden content. For instance, a user might think they are clicking a 'Play' button on a video, but they are actually clicking a button that triggers an unwanted action, such as changing their account settings or authorizing a transaction. This technique exploits the trust a user has in a specific website and can lead to unauthorized actions, such as sharing confidential information or manipulating user accounts. To mitigate clickjacking attacks, web developers can implement frame-busting techniques, use the X-Frame-Options header to control how their content is displayed in iframes, and educate users about potential risks.

Examples

  • A user clicks on what they believe is a play button for a video, but it actually authorizes a payment.
  • A user is tricked into changing their privacy settings on a social media platform by clicking on a disguised button.

Additional Information

  • Clickjacking is often used in conjunction with social engineering attacks.
  • Web browsers have introduced various security measures, like Content Security Policy (CSP), to help prevent clickjacking.
