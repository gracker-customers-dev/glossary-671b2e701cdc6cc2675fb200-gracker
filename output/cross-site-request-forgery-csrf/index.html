Cross-site Request Forgery (CSRF)

A type of attack that tricks a user into executing unwanted actions on a web application in which they are authenticated.

Description

Cross-site Request Forgery (CSRF) is a security vulnerability that allows an attacker to induce users to perform actions they do not intend to perform on a web application where they are currently authenticated. This type of attack exploits the trust that a web application has in the user's browser. When a user is logged into a site, their browser stores session cookies. If the user unknowingly visits a malicious site while still logged in, the attacker can craft a request to the target web application that is sent using the user's credentials. For example, if the user is authenticated to a banking site, the attacker could potentially transfer funds without the user's consent. CSRF attacks can lead to unauthorized actions like changing account settings, initiating transactions, or even compromising data integrity. Mitigating CSRF attacks typically involves implementing anti-CSRF tokens, same-site cookie attributes, and ensuring that sensitive actions require additional user verification.

Examples

  • A user is logged into their online banking account and visits a malicious website that submits a fund transfer request without their knowledge.
  • A user is tricked into clicking a link in an email that changes their profile information on a web application where they are authenticated.

Additional Information

  • CSRF tokens are unique, secret, and unpredictable values generated by the server and included in requests to verify authenticity.
  • Modern browsers implement security measures like the SameSite cookie attribute to help mitigate CSRF risks.
